# Лабораторная работа #2
## Microservices

### Формулировка

В рамках второй лабораторной работы требовалось реализовать систему, состоящую из нескольких
взаимодействующих друг с другом сервисов.

### Требования

1. Каждый сервис имеет свое собственное хранилище, если оно ему нужно. Можно использовать один
   instance базы данных но каждый сервис работает _только_ со своей логической базой. Запросы между базами _запрещены_.
2. Для межсервисного взаимодействия использовался HTTP (RESTful).
3. Выделен **Gateway Service** как единая точка входа (без горизонтальных запросов между
   сервисами).
4. На каждом сервисе сделан специальный endpoint `GET /manage/health`, отдающий 200 ОК.
6. Сборка производится при помощи Github Actions.
7. Gateway Service запускается на порту 8080, остальные сервисы на портах 8050, 8060, 8070.
8. Каждый сервис завернут в docker.
9. В [docker-compose.yml](docker-compose.yml) прописана сборка и запуск docker контейнеров.
10. В [classroom.yml](.github/workflows/classroom.yml) дописаны шаги на сборку и прогон unit-тестов.
11. Для автоматических прогонов тестов в файле [autograding.json](.github/classroom/autograding.json)
    и [classroom.yml](.github/workflows/classroom.yml) заменен на v2 (Вариант с отелем).
